# v6 — философия

**Аудитория:** LLM, читающий этот репозиторий впервые или повторно. Не человек.

Документ отвечает на **зачем**. На **как** — `AGENTS.md`, `architecture.yaml`, `CHECK.md`, и per-package `SPEC.md` / `README.md`.

---

## В одной строке

v6 — реактивная TypeScript-экосистема, спроектированная так, чтобы **LLM-агент был её первоклассным потребителем**, равным человеку-разработчику.

---

## Что это значит для тебя как LLM

1. **Точка входа одна: `alaq`.** Человек пишет `npm install alaq` и получает всё. Ты, зайдя в новый проект, запускаешь `alaq` и получаешь **capability manifest** — машиночитаемый JSON с описанием экосистемы: какие пакеты доступны, какие MCP-инструменты у тебя на руках, что ты можешь делать.

2. **MCP — твой главный интерфейс.** `@alaq/mcp` — stdio JSON-RPC сервер. Через него ты **компилируешь SDL** (`schema_compile`), **анализируешь изменения** (`schema_diff` — с классами `breaking` / `non_breaking` / `review`), **наблюдаешь живой рантайм** (7 `alaq_*` tools поверх Logi). Без MCP ты скрапишь stdout и гадаешь; через MCP — получаешь структурированный ответ.

3. **Диагностики — стабильный JSON.** Коды ошибок (`E001`–`E022`, `W001`–`W004`) не меняются между minor-версиями. Ты можешь завязывать автоматику на конкретный код.

4. **Sandbox реален.** `rootDir` в filesystem-режиме MCP — не декорация. Попытка `../etc/passwd` отклоняется. Ты можешь доверять.

5. **Категория `review` в diff-классификаторе — твоё право остановиться.** Когда классификатор говорит `review`, это сигнал: "решение требует осмысления, не автоматики". Не пропускай его молча.

## Семь принципов (нормативно)

### 1. SDL — единственная истина (относится к alaqlink)
`.aql` пишется один раз. Типы, wire, CRDT, хендлеры, композаблы — генерируются. Закрытый набор директив в `packages/graph/SPEC.md §7`.

### 2. Core нейтрален, плагины именуются по транспорту
`@alaq/graph` не знает про Zenoh/Vue/Tauri. Генератор именуется по **мишени** (`graph-zenoh`, `graph-link-state`), **не** по продукту (`graph-kotelok` — запрещено). Продукты — потребители.

### 3. Типы сильнее рантайм-проверок
Что компилятор доказал — рантайм не валидирует. Валидация только на границе (входящие сообщения).

### 4. AI — первоклассный потребитель
Это не принцип-лозунг, а физическое свойство стека. MCP-сервер, машиночитаемые диагностики, стабильные коды ошибок, capability manifest в `alaq`, `review`-категория в diff — всё ради того, чтобы LLM работал без человека-посредника.

### 5. Scopes, а не singleton
`@scope(name: "room")` — один инстанс на именованный скоуп. Клиент подписывается на `room/<id>`, сервер маршрутизирует. Ты получаешь **кусок** состояния, не весь объект.

**Границы (нормативно, см. `packages/graph/SPEC.md §7.5`):**
- **Scope одномерен by design.** Запись опирается ровно на один scope. Multi-axis срезы данных (например `channel × region × user`) не выражаются стэком `@scope`; главная ось — scope, остальное — обычные аргументы `input` плюс server-side фильтрация в хендлере. Если тянет написать `@scope(channel, admin)` — ответ «нет»: выбери главную ось, остальные протащи аргументами.
- **Auth — не scope.** `@scope` управляет реактивным срезом и жизненным циклом, никогда не авторизацией. Аутентификация и авторизация живут на **транспортном слое** — HTTP-middleware для `graph-axum`, Tauri capabilities / per-command проверки для `graph-tauri-rs`, per-topic ACL в рантайме для `graph-zenoh`. SDL не описывает auth. Actions без `scope` — валидная форма для admin-мутаций; кто может их вызвать, решает конфиг генератора или middleware.
- **Транспорт — не scope.** По чему именно идёт action — HTTP, Tauri IPC или Zenoh — решает **какой генератор потребляет IR**, а не `@scope`. См. out-of-scope список в `SPEC.md §17`.

### 6. Tier-based транспорт
T0: ws+http. T1: +webrtc. T2: +zenoh. T3: zenoh-wasm. Tree-shaking — механизм enforcement. Поднимаешься по требованиям, не наоборот.

### 7. Запреты
- Нет runtime-директив в `@alaq/graph` (живут в плагинах).
- Нет TS AST для извлечения схем (схема — в `.aql`).
- Нет cross-import между генераторами.
- Нет `graph-<productname>` пакетов.

## Что v6 **не делает**

- Не управляет identity / auth. Слой выше.
- Не описывает deployment.
- Не предписывает UI-фреймворк.
- Не пытается быть GraphQL. Read-паттерны — другая задача.
- Не встраивает LLM. alaq — фронтдор к инструментам, сам не думает.
- Не валидирует в рантайме то, что компилятор уже доказал.

## Соотношение со смежными инструментами

| Инструмент        | Где пересекается         | Где нет                                          |
|-------------------|--------------------------|--------------------------------------------------|
| GraphQL           | типизированный контракт  | v6 — codegen+sync, не query                      |
| gRPC / protobuf   | wire + IDL               | v6 — реактивные подписки, не вызов-ответ         |
| tRPC              | typesafe RPC             | v6 не привязан к TS на обоих концах              |
| Yjs / Automerge   | CRDT                     | v6 даёт типизированный SDL поверх CRDT           |
| Effect Schema     | валидация + типы         | v6 — wire + транспорт + runtime                  |
| Sentry            | observability            | Logi (через `@alaq/plugin-logi`) — MCP-first    |

## Имена

- **`alaq`** — npm-имя фронтдора (без scope). `npm install alaq`.
- **`@alaq`** — npm-scope для всех индивидуальных пакетов (`@alaq/graph`, `@alaq/mcp`, ...).
- **`v6`** — общее имя экосистемы и major-цикла. Не продукт, а эпоха.
- **`alaqlink`** — трек внутри v6 (L2): SDL + генераторы + runtime sync. Не синоним v6.

Если видишь "alaqlink" — ищи в L2. Если видишь "v6" — думай обо всей экосистеме.

## Стабильность

Сейчас `6.0.0-alpha.0`. До `6.0.0` GA поверхность меняется: имена tools, форма IR, набор директив. Семантика SDL в `SPEC.md` v0.3 стабильна.

После GA — строгий semver на публичные API. До GA — breaking помечаются в CHANGELOG, не молчаливо.

## Куда идти дальше

Если ты — LLM, впервые вошедший в этот репо:

1. **Прочитай `AGENTS.md`** — нормативные правила твоего поведения.
2. **Запусти `CHECK.md`** (или прочитай, если не можешь выполнять) — протокол верификации состояния.
3. **Открой `architecture.yaml`** — машиночитаемый реестр, по нему навигируй.
4. **Если тебе подключён MCP** — первый вызов: `alaq_capabilities`. Он расскажет, что у тебя в руках.
5. **Если MCP не подключён** — запусти `alaq mcp install` чтобы получить стандартный MCP-stanza, скажи человеку вставить его в конфиг своего MCP-клиента, потом повтори шаг 4.
