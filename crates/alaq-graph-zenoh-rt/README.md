# alaq-graph-zenoh-rt

Runtime support for code emitted by `@alaq/graph-zenoh`. Wraps `automerge::AutoCommit` into a `CrdtDoc` type consumed by generator-emitted `<Doc>Doc` wrappers for composite CRDT documents (multi-map root layout with a pinned `schema_version`). The API is **fully monomorphic**: callers hand in already-serialised JSON strings (`upsert_json` / `list_json`) — generic Serde bounds live in the generated wrappers, not here, so the crate stays dispatch-free and `dyn`-compatible. Current status: **API contract v0.1 (SPEC 0.3.7), implementations TBD** — every method body in `src/lib.rs` is `unimplemented!()` awaiting the follow-up implementation wave. `upsert_json` takes an explicit `lww_field` so the runtime can update the LWW resolution cell alongside the JSON blob (SPEC §7.15 R234); `delete_entry` takes `Option<SoftDeleteSpec>` — `None` = hard delete, `Some { flag, ts_field }` = tombstone-by-flag (R235), matching Busynca's `SyncPoint.is_deleted` convention byte-for-byte. The wire layout is documented in `@alaq/graph` SPEC §11 (composite Automerge mapping) and §7.15 R232 (entry values are JSON strings inside the Automerge cells, for byte-parity with Busynca's `GroupSyncManager`). Consumers of this crate are the generated `<Doc>Doc` wrappers — application code should call those, not the low-level `CrdtDoc` methods directly. Do not change the API signatures without cross-agent coordination — the graph-zenoh codegen depends on this exact surface.
