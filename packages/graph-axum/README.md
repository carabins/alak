# @alaq/graph-axum

Generator: compiles `.aql` IR into a set of Rust files — serde types, an
`#[async_trait] trait Handlers`, an axum `Router`, and an `AppState<H>`
wrapper. For **server-side** consumers using [`axum`](https://docs.rs/axum).

## Status

`6.0.0-alpha.0` — **unstable**. Output shape, method conventions, and
`GenerateOptions` move between alpha releases.

## What it emits

One subdirectory per namespace, containing five `.rs` files:

```
rest_valkyrie_arsenal/
├── mod.rs         pub mod + re-exports
├── types.rs       records, enums, <Action>Input
├── handlers.rs    #[async_trait] pub trait Handlers
├── state.rs       pub struct AppState<H> + manual Clone
└── routes.rs      pub fn router<H: Handlers>(state) -> Router
                   + per-action dispatch_<action><H>
```

### Conventions (normative for v0.1)

- **Flat namespace.** `rest.valkyrie.arsenal` → `rest_valkyrie_arsenal/`.
  Non-alphanumerics collapse to `_` so the path is Rust-safe.
- **snake_case everywhere.** Field names, action method names, and HTTP paths
  are all snake (matches `@alaq/graph-tauri-rs` / `@alaq/graph-tauri`, see
  stress-test C.1/C.2). No `#[serde(rename_all = "camelCase")]`.
- **One HTTP verb.** Every action → `POST /<snake_action_name>`. Input is a
  JSON body; output is `200 OK` + JSON for actions with declared output, or
  `202 Accepted` with an empty body for fire-and-forget actions.
- **Typed errors.** Handlers return `Result<T, HandlerError>` from the
  runtime crate. `HandlerError::IntoResponse` maps variants to HTTP statuses
  (`BadRequest` → 400, `NotFound` → 404, `Unauthorized` → 401,
  `Forbidden` → 403, `Internal` → 500) with a stable JSON body
  `{ "error": "...", "code": "..." }`.
- **Trait DI.** The user implements `Handlers` once per application and
  passes `Arc<H>` via `AppState::new`. The router is generic over `H`.
- **Admin / auth is OUT of SDL.** `@scope` on records triggers a warning.
  Enforce authorization with an axum middleware layered on top of the
  generated `router(...)`.
- **Output typing — inline, no newtype (C1, 2026-04-21).** The handler
  trait and dispatchers spell the output Rust type directly; no
  `<Action>Output` alias/newtype is emitted.
  - list output (`[T!]!`) → `Vec<T>` (and `Vec<Option<T>>` / `Option<Vec<…>>`
    variants when the outer/inner is nullable)
  - scalar output (`T!`)   → `T` (or `Option<T>` when not required)
  - no output              → `()`, dispatcher returns 202
  A `#[serde(transparent)]` Vec newtype is wire-identical to a bare Vec,
  so the removal is a pure cleanup — zero wire impact, one fewer type per
  action.
- **Input typing.** `<Action>Input` is always emitted, even when SDL declares
  no input fields (a `Default`-able empty struct). Keeps handler signatures
  uniform.

### What it does NOT do

- No events / streams / subscriptions (HTTP is request/response).
- No directive-aware codegen beyond preserving scope as a warning.
- No Cargo.toml generation — the consumer wires it up manually.
- No real-world `Handlers` impl — you write that.

## Install

```sh
bun add -d @alaq/graph-axum @alaq/graph
```

The generated Rust code expects these crates in the consumer's `Cargo.toml`:

```toml
axum = { version = "0.8", features = ["json"] }
serde = { version = "1", features = ["derive"] }
serde_json = "1"
alaq-graph-axum-rt = { path = "<workspace>/crates/alaq-graph-axum-rt" }
# plus tokio for the binary
```

## Quickstart

```ts
import { compileSources } from '@alaq/graph'
import { generate } from '@alaq/graph-axum'
import { readFileSync, writeFileSync, mkdirSync } from 'node:fs'
import { join, dirname } from 'node:path'

const src = readFileSync('schema/arsenal.aql', 'utf8')
const { ir, diagnostics } = compileSources([{ path: 'arsenal.aql', source: src }])
if (!ir) throw new Error(diagnostics.map(d => d.message).join('\n'))

const out = generate(ir)
for (const f of out.files) {
  const p = join('generated', f.path)
  mkdirSync(dirname(p), { recursive: true })
  writeFileSync(p, f.content)
}
```

Then in your Rust crate:

```rust
mod generated;
use std::sync::Arc;
use generated::rest_valkyrie_arsenal::{router, AppState, Handlers};

struct App;

#[alaq_graph_axum_rt::async_trait]
impl Handlers for App {
    // ... one async fn per action
}

#[tokio::main]
async fn main() {
    let state = AppState::new(Arc::new(App));
    let app = router(state);
    let listener = tokio::net::TcpListener::bind("0.0.0.0:8099").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}
```

## `GenerateOptions`

```ts
export interface GenerateOptions {
  namespace?: string        // default: emit every namespace
  header?: boolean          // default: true — emit AUTOGENERATED banner
  runtimeCrate?: string     // default: "alaq-graph-axum-rt"
                            //          (also accepts underscore form)
}
```

## Consuming generated code

### Runtime dependency

The generated tree expects `alaq-graph-axum-rt` in your `Cargo.toml` (see
the Install section above) — it provides `HandlerError`,
`ActionContext::from_request_parts`, and the `async-trait` re-export.
`HandlerError::IntoResponse` gives you a stable
`{ "error": "...", "code": "..." }` JSON body with the status codes listed
in the Conventions section.

### Windows `#[path]` edge case (О20)

**Normative pattern: flat `#[path]` at the crate root, not a nested
`mod generated { #[path = "..."] pub mod ns; }` wrapper.**

Generated code lives outside `src/` (typically at
`schema/generated/rs/<namespace>/`). The common instinct is to wrap it in
a decorative `mod generated`:

```rust
// ❌ Broken on Windows.
mod generated {
    #[path = "../../schema/generated/rs/rest_valkyrie_arsenal/mod.rs"]
    pub mod rest_valkyrie_arsenal;
}

use generated::rest_valkyrie_arsenal::{router, AppState, Handlers};
```

`rustc` resolves the nested `#[path]` relative to a virtual `src/generated/`
prefix — the `..` segments never climb above that virtual root, so the
physical path `server-v2\src\generated\..\..\schema\generated\...` is
followed and does not exist. On Windows this surfaces as
`os error 3` ("path not found"); on Unix you get a similar ENOENT.

**Use a flat `#[path]` on the crate root instead:**

```rust
// ✅ Works on Windows and Unix.
#[path = "../../schema/generated/rs/rest_valkyrie_arsenal/mod.rs"]
mod rest_valkyrie_arsenal;

use rest_valkyrie_arsenal::{router, AppState, Handlers};
```

The inner `pub mod types;` / `pub mod handlers;` / `pub mod state;` /
`pub mod routes;` in the generator's own `mod.rs` resolve correctly once
the parent is found. Import at the crate root uses the bare namespace
name — no `generated::` prefix.

If you want a `generated::` grouping for stylistic reasons, the workable
alternative is a `build.rs` that copies or `include!`s the generated
files into `OUT_DIR`, then a single `mod generated { include!(...); }`
at build time. More moving parts — prefer the flat `#[path]` unless you
need it.

### Re-generation hygiene

`generate(ir, options)` is deterministic: actions/records/enums are
`Object.keys(...).sort()`-ed before emission. Byte-for-byte reproducible
between runs — snapshot tests in `test/snapshots/` are stable. Commit the
build-script (`schema/_generate_axum.ts` or equivalent), not the
generated `.rs` files (or commit both and enforce `bun run generate &&
git diff --exit-code` in CI).

### Pair with `@alaq/graph-tauri-rs`

For the same Rust crate, you can pair the axum router with a Tauri-side
generator using [`@alaq/graph-tauri-rs`](../graph-tauri-rs). Both follow
the snake_case method-name / `<Action>Input` struct conventions, so a
schema split between HTTP-facing actions and IPC-facing actions (two
`.aql` files) emits compatible Rust that can share `Handlers`
implementations. Output types are spelled inline on both sides (see
C1 cleanup above).

## Open design questions

Tracked in [`../../stress.md`](../../stress.md):

- **О20** — documented above (Windows `#[path]` nested-mod edge case and
  the flat-path pattern). Open: should the generator emit a
  `// integration-note:` comment in the top of `mod.rs` linking to this
  README section, or is the README enough?
- **О21** — `GENERATOR_VERSION` const drifts from `package.yaml` version
  across sister generators (`graph-axum`, `graph-tauri`,
  `graph-tauri-rs`). Auto-generate on build vs manual until out of
  alpha/draft.
