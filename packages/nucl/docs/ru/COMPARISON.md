# Сравнение: Nucl vs Quark vs Vue

Этот документ помещает `@alaq/nucl` в контекст экосистемы.

## 1. Nucl vs Quark

**Quark** — это двигатель. **Nucl** — это автомобиль.

| Характеристика | Quark | Nucl |
| :--- | :--- | :--- |
| **Философия** | "Металлическая" Производительность | Удобство Разработки (DX) |
| **API** | Минимальный (.up, .value) | Расширяемый (Плагины) |
| **Глубокая Реактивность** | Нет (Только примитивы/ссылки) | Да (через `deep-state`) |
| **Computed** | Ручное (Pipe/Listeners) | Первоклассная поддержка (`fusion`) |
| **Скорость создания** | Экстремальная (~12k ops/ms) | Очень высокая (~6.5k ops/ms) |
| **Память** | Крошечная | Малая (Оверхед прототипов) |

**Когда использовать Quark?**
- Высокочастотный трейдинг, Игры (цикл 60fps), массивная визуализация данных.
- Когда вам не нужны хелперы вроде `.push()` или computed properties.

**Когда использовать Nucl?**
- Управление состоянием приложения (Stores).
- Логика UI (где часты вычисляемые значения).
- Когда нужны удобные методы (`isEmpty`, `assign`).

## 2. Nucl vs Vue Reactivity (`ref`)

Nucl имитирует API и эргономику `ref`, но фокусируется на явном потоке данных.

| Характеристика | Vue `ref` | Nucl |
| :--- | :--- | :--- |
| **Трекинг зависимостей** | Автоматический (Глобальный Граф) | Явный (Edges/Fusion) |
| **Глубокая Реактивность** | Включена по умолчанию | Опциональна (Плагин) |
| **Модульность** | Фиксированный набор | Компонуемый (Плагины) |
| **Производительность** | Оптимизировано для DOM | Оптимизировано для Data Flow |
| **Межмодульная связь** | Требует context/props | Нативная (Realms) |

**Ключевое отличие:**
Реактивность Vue — "Магическая" (она следит за тем, что вы читаете).
Реактивность Nucl — "Явная" (вы декларируете, от чего зависите).

Явная реактивность (Nucl/Fusion) обычно:
1.  **Более предсказуема:** Нет загадок "почему это перерисовалось?".
2.  **Более производительна:** Нет оверхеда на обход графа при чтении.
3.  **Сложнее в написании:** Вы должны перечислять зависимости вручную.

## Резюме

- Используйте **Quark** для сырых примитивов.
- Используйте **Nucl** для состояния приложения и логики.
- Используйте **Vue Ref** внутри компонентов Vue (для удобства связывания с DOM).
