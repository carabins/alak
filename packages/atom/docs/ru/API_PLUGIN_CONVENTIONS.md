# Плагин `ConventionsPlugin`

Плагин внедряет соглашения по именованию методов, позволяя декларативно подписываться на изменения свойств и события шины данных.

## Соглашения

### 1. Подписка на свойство (`_prop_up`)
Любой метод, имя которого начинается с подчеркивания и заканчивается на `_up`, будет автоматически подписан на соответствующее свойство.

```typescript
class User {
  status = 'offline';

  // Вызовется автоматически при изменении status
  _status_up(val: string) {
    console.log('User status is:', val);
  }
}
```

### 2. Подписка на события шины (`_on_Event`)
Любой метод, имя которого начинается с `_on_`, будет автоматически подписан на соответствующее событие в Quantum Bus текущего Realm.

```typescript
class NotificationModel {
  // Подписка на глобальное событие ALERT
  _on_ALERT(payload: any) {
    alert(payload.message);
  }
}
```

## Использование с `IAtomic`
Чтобы получить автокомплит для имен этих методов, используйте наследование от `IAtomic`:

```typescript
import { IAtomic } from '@alaq/atom';

class MyStore extends IAtomic<MyStore> {
  count = 0;
  
  // IDE предложит это имя при наборе
  _count_up(val: number) { ... }
}
```

## Преимущества
- **Чистота конструктора**: Не нужно вручную вызывать `.up()` или `.on()`.
- **Автоматическая очистка**: Все подписки, созданные плагином, отключаются при вызове `atom.$.decay()`.
