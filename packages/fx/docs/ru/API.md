# Справочник API

## Ядро (Core)

### `fx(source)`

Создает новую цепочку эффекта на основе подписываемого источника.

**Аргументы:**
- `source`: Объект, реализующий интерфейс `Subscribable<T>`.

**Возвращает:**
- Экземпляр строителя `Fx<T>`.

**Пример:**
```typescript
import { fx } from '@alaq/fx';
fx(myAtom).up(val => console.log(val));
```

### `.up(handler)`

Подписывается на цепочку и активирует эффект. Это терминальная (последняя) операция.

**Аргументы:**
- `handler`: `(value: T) => void` - Функция, выполняемая при срабатывании эффекта.

**Возвращает:**
- `Unsubscribe`: Функция для остановки эффекта и очистки ресурсов (таймеров, слушателей).

**Пример:**
```typescript
const stop = fx(count).up(c => console.log(c));
// Позже
stop();
```

## Трансформации

### `.map(fn)`

Преобразует значение, проходящее через цепочку.

**Аргументы:**
- `fn`: `(value: T) => R` - Функция проекции.

**Возвращает:**
- `Fx<R>`

**Пример:**
```typescript
fx(user).map(u => u.name).up(updateHeader);
```

### `.with(otherSource)`

Считывает ("семплирует") текущее значение другого источника **без подписки** на его изменения.
Полезно для добавления контекста (payload) к событию.

**Аргументы:**
- `otherSource`: `Subscribable<S>`

**Возвращает:**
- `Fx<[T, S]>` - Кортеж [оригинальноеЗначение, другоеЗначение]

**Пример:**
```typescript
fx(submitClick)
  .with(formData)
  .up(([_, data]) => submitForm(data));
```

## Фильтры (Gates)

### `.filter(predicate)` / `.when(predicate)`

Останавливает распространение события, если предикат возвращает `false`.

**Аргументы:**
- `predicate`: `(value: T) => boolean`

**Возвращает:**
- `Fx<T>`

**Пример:**
```typescript
fx(age).when(a => a >= 18).up(grantAccess);
```

### `.skip(n)`

Пропускает первые `n` событий.

**Пример:**
```typescript
fx(source).skip(1).up(fn); // Игнорировать начальное значение
```

### `.take(n)`

Принимает только первые `n` событий, после чего автоматически отписывается.

**Пример:**
```typescript
fx(source).take(1).up(fn); // Работает как .once()
```

## Тайминг (Timing)

### `.debounce(ms)`

Откладывает выполнение до тех пор, пока источник не перестанет генерировать события в течение `ms` миллисекунд.
Идеально для обработки быстрого ввода (печать, ресайз окна).

**Аргументы:**
- `ms`: `number` - миллисекунды ожидания "тишины".

**Пример:**
```typescript
fx(input).debounce(300).up(saveToDb);
```

### `.delay(ms)`

Сдвигает выполнение на `ms` миллисекунд.
Если источник выдает новое значение во время ожидания, предыдущее отложенное событие отменяется.
Идеально для предотвращения мигания UI (например, показывать лоадер только если запрос идет долго).

**Аргументы:**
- `ms`: `number`

**Пример:**
```typescript
fx(loading).when(v => v).delay(500).up(showSpinner);
```

## Асинхронность (Async)

### `.async(handler)`

Выполняет асинхронную операцию.
Автоматически управляет **конкурентностью**: если новое значение приходит, пока предыдущая асинхронная операция еще не завершена, предыдущая отменяется (abort), реализуя паттерн "Switch Map".

**Аргументы:**
- `handler`: `(value: T, signal: AbortSignal) => Promise<R>`

**Возвращает:**
- `Fx<R>` - Поток продолжается с результатом промиса.

**Пример:**
```typescript
fx(userId)
  .async(async (id, signal) => {
    // signal.aborted станет true, если userId изменится снова
    const response = await fetch(`/users/${id}`, { signal });
    return response.json();
  })
  .up(userData => renderProfile(userData));
```

## Интерфейс `Subscribable<T>`

Чтобы любой объект был совместим с `fx`, он должен реализовать:

```typescript
interface Subscribable<T> {
  up(listener: (val: T) => void): any;    // Подписка
  down(listener: (val: T) => void): void; // Отписка
  value: T;                               // Доступ к текущему значению
}
```
