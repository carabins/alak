# @alaq/fx

> **Zero-dependency**, универсальная система реактивных эффектов для управления сайд-эффектами, асинхронностью и таймингами.

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**fx** предоставляет удобный цепочный (fluent) API для превращения потока данных в действия. Библиотека спроектирована для работы с `@alaq/quark`, `@alaq/nucl` или любым другим объектом, реализующим интерфейс `Subscribable`.

[🇺🇸 Read in English](./README.md)

## Особенности

- 🎯 **Агностичность**: Работает с любым стейт-менеджером (Quark, Nucleon, Atom или кастомные решения).
- ⛓️ **Fluent API**: Цепочки операторов (`map`, `filter`, `debounce`, `async`).
- ⏱️ **Управление временем**: Встроенные `debounce` и `delay` с автоматической очисткой таймеров.
- ⚡ **Асинхронность**: Поддержка `AbortSignal` из коробки и защита от гонки состояний (авто-отмена старых запросов).
- 🪶 **Легковесность**: Ноль runtime-зависимостей.

## Быстрый старт

```typescript
import { fx } from '@alaq/fx';
import { Qv } from '@alaq/quark'; // или любой другой источник

const searchQuery = Qv('');

fx(searchQuery)
  .map(q => q.trim())           // 1. Преобразование
  .filter(q => q.length >= 3)   // 2. Фильтрация
  .debounce(300)                // 3. Ожидание ввода
  .async(async (q, signal) => { // 4. Запрос с отменой
     const res = await fetch(`/api/search?q=${q}`, { signal });
     return res.json();
  })
  .up(results => {              // 5. Подписка и действие
    console.log('Результаты:', results);
  });
```

## Философия

Реактивные примитивы (Атомы, Сигналы) хранят **Состояние**.
**fx** управляет **Потоком** и **Побочными эффектами**, возникающими при изменении этого состояния.

Вместо императивных `subscribe` колбэков, наполненных `clearTimeout` и проверками на актуальность данных, `fx` позволяет декларативно описать *что* должно произойти и *когда*.
